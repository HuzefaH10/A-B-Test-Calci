Running Best Practices
  • Avoid early stopping: Do not end the test early even if statistical significance is reached prior to hitting the required sample size of 0 per variant. Doing so increases the risk of false positives.
  • Keep daily volumes equal: Try to route 50% of traffic to Control and 50% to Variation to minimize test runtime.
  • Full-week cycles: Always run tests in full-week multiples (e.g., 7, 14, 21 days) to account for weekly traffic pattern anomalies (e.g., weekend behavior differences).
